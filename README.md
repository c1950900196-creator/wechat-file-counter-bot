# 微信群文件统计机器人

> 自动统计微信群内的文件数量，未达标时自动发送提醒。适用于需要定期收集文件的工作群（如会议材料、报告提交等场景）。

**本项目由产品经理使用 Cursor + AI 辅助开发，零手写代码。**

## 功能亮点

- **自动统计** — 实时监听群消息，自动统计附件、图片、视频数量
- **多群管理** — 支持同时监控多个群，每个群独立设置目标数量
- **定时提醒** — 支持 Cron 表达式配置提醒时间，未达标自动提醒
- **关键词查询** — 群内发送"统计"即可查看当日统计结果
- **Web 管理界面** — 浏览器配置群列表、目标数量、提醒时间
- **扫码登录** — Web 界面展示二维码，手机扫码即可上线

## 技术栈

| 模块 | 技术 |
|------|------|
| 运行时 | Node.js ≥ 16 |
| 微信协议 | Wechaty + wechaty-puppet-wechat4u |
| 定时任务 | node-schedule |
| Web 服务 | 原生 HTTP 模块 |
| 数据存储 | JSON 文件 |

## 快速开始

```bash
# 1. 安装依赖
npm install

# 2. 启动（带 Web 管理界面）
npm start

# 3. 打开浏览器 http://localhost:3000 扫码登录
```

## 使用方式

1. 启动后打开 `http://localhost:3000`，用手机微信扫码登录
2. 在 Web 界面添加需要监控的群名和目标文件数
3. 设置每日提醒时间（如 18:00）
4. 机器人自动统计并在设定时间发送提醒

## 群内命令

| 关键词 | 功能 |
|--------|------|
| 统计 / 文件统计 | 查看当日文件统计 |
| 今日统计 | 查看当日统计详情 |
| 查询 | 查看当前统计状态 |

## 注意事项

- 使用非官方微信协议，建议使用小号登录
- 首次扫码后会话会保存到本地，下次启动无需重新扫码
